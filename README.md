# Tempo_Comments
Jira Tempo Auto Comments

Автоматическое запомление Jira Tempo на основе комментариев в задачах от сотрудников.

## Цель скрипта

- Взять на себя рутину и разгрузить сотрудников Компании.
- В среднем, сотрудник тратит до 30 минут на заполнение ворклогов в день!!!

## Условия:
- В идеале сотрудник в начале комментария указывает название продукта и время, которое нужно списать.

  Например,
  ```
  Самый лучший продукт в Мире
  25 минут
  Проводились работы по тестированию продукта. Совместно с архитектором добавили Pull Request в репозиторий
  ```

  Используется регулярка. Скрипт сам подберет нужное время и найдет продукт, даже если он на латинице.

- Запуск скрипт регламентируется настройками в .env.
- Все настройки вынесены в .env
- Все чувствительные данные вынесены в .env

- Учитывается время на ежедневные встречи.
- Если сотруник во время дейли был в проекте, то дейли не проставляется.
- Учитывается время еженедельных встреч (в понедельник).

- Учитываются команды.
- Можно исключать сотрудников из автоматизации. Перед его никнеймом потребуется поставить ! (!username)
- Учитывается разное написание учетных записей в Jira и в Jira Tempo.

- Добавлена обязательная проверка дублей ворклогов.
- В комментарии в ворклоге от скрипта (Автоматизация) ставится кодовый эмодзи. AUTO_TAG=🤖

## Установка

- **Склонировать репозиторий**
```bash
git clone https://github.com/adjuster2004/Tempo_Comments/
cd Tempo_Comments
cp .env.example .env
```

- **Заполнить все необходимые поля в .env**

Обязательно нужно заполнить верхнюю часть файла, где указываются подключения.

```bash
# Доступы к Jira
JIRA_BASE_URL=https://FQDN
JIRA_USER_EMAIL=
JIRA_API_TOKEN=

# Доступы к Tempo (часто требуется отдельный токен, в зависимости от версии)
TEMPO_API_TOKEN=

# Настройки скрипта. Указываем сотрудников или команды через запятую
# Если нужно исключить пользователя, пишем в TARGET_USERS !username

TARGET_USERS=

TARGET_TEAMS=

# указываем кодовые названия проектов Jira
TARGET_PROJECTS=
AUTO_TAG=🤖
```

По остальным полям, если сомневаетесь, просто оставьте пустыми.

- **Запуск скрипта в контейнере**
  
```bash
docker-compose up -d --build
```

- **Просмотр логов**
  
```bash
docker logs jira-tempo-sync
```

- **Принудительный запуск обработки комментариев в Jira задачах**
  
```bash
docker exec -it jira-tempo-sync python /app/comment.py
```

- **Вывод соответствия данных поля продукта в Jira задачах и в Tempo Jira**
  
```bash
docker cp get_attrs.py jira-tempo-sync:/app/get_attrs.py
docker exec -it jira-tempo-sync python /app/get_attrs.py
```

- **Или полный вывод соответствия**
  
```bash
docker cp get_attrs.py jira-tempo-sync:/app/get_dict_full.py
docker exec -it jira-tempo-sync python /app/get_dict_full.py
```

- **Остановить контейнер**
  
```bash
docker compose down
```

## Запуск без compose

```bash
docker rm -f jira-tempo-sync
docker build -t jira-tempo-sync .
docker run -d --name jira-tempo-sync --env-file .env --restart unless-stopped jira-tempo-sync
```
